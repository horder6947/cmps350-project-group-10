Description of the project:
You will build a social media platform like Twitter, Instagram, TikTok etc. This document outlines the requirements for the first phase 
of the social media platform. The goal is to build a user-friendly and visually appealing interface that is both functional and responsive.
• There is a single user type: Users who post messages and follow others, and do other activities.
• Each user has a user id (unique for each user), user name (shown in their profile), email address, password, profile picture.

Core Features
These are the essential features that the front-end must support.

Feature Description
User Authentication Users should be able to create an account and log in. This includes a registration 
form with validation (e.g., for email format and password strength) and a login 
form.
User Profile A dedicated page for each user, displaying their information (e.g., username, bio) 
and their posts. The logged-in user should be able to edit their own profile.
News Feed The main page of the platform, displaying a stream of posts from the users they 
follow. Each post should show the author's username, the content of the post, and 
the timestamp.
Create/Delete and View Posts
Users must be able to create new posts (text-based to start) or delete them. They 
should also be able to view a single post in more detail, along with its comments.
Like and Comment on Posts
Users should be able to "like" a post, and the like count should update. They should 
also be able to add comments to a post.
Follow/Unfollow Users
Users should be able to follow and unfollow other users to see their posts in the 
news feed.

Technical Stack & Constraints
This is the required technology stack. No external libraries or frameworks are permitted.

Technology Requirement
HTML5 Use semantic HTML5 to give the page proper structure and meaning 
(<nav>, <main>, <section>, etc.).
CSS3 Use Flexbox and Grid to create all layouts. The application must be responsive and adapt 
to different screen sizes purely with CSS. No CSS frameworks like Bootstrap are allowed.
Vanilla JavaScript (ES6+)
All interactivity must be built with plain JavaScript. This includes: - DOM Manipulation 
(creating/updating/deleting elements) - Event Handling (listening for clicks, form 
submissions) - Managing all application data locally.
Git & GitHub The entire project must be managed with Git for version control and hosted on a public or 
private GitHub repository.

Data Handling
Since there is no back-end, all data must be stored and managed within the browser.

Requirement Description
Local Storage Use the browser's localStorage API to make the data persist between page reloads and 
browser sessions.
JSON Format All data (user accounts, posts, likes, comments) should be structured and stored 
in localStorage as JSON. You will need to use JSON.stringify() to save the data 
and JSON.parse() to retrieve it.
Data Structure Plan a clear structure for your JSON data from the start. For example, you might have top level arrays for users and posts.

4. How to Use Your Application (A Quick Guide)
This is the most important part of the report. Provide a simple, step-by-step guide on how to test the main functionalities of your 
website.

Example:
1. Register a New User:
• Navigate to the Register page.
• Enter a username and a simple password, then click "Register".
• You will be redirected to the Login page.
2. Log In and Create a Post:
• On the Login page, enter the credentials you just created.
• You will be taken to the News Feed.
• Use the "What's on your mind?" text box at the top to write a post and click "Post". Your new post will appear at the 
top of the feed.
3. Interact with Posts:
• Click the "Like" button on any post to see the count increase.
• Type a comment in the comment box under a post and click "Comment" to add it.
